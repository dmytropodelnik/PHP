Разработка API

Работу API обеспечивают скрипты PHP, рекомендуется
обращаться к ним через маршрутизатор (.htaccess).
Это позволит:
	a) скріть расширение (технологию);
	б) убедиться в работоспособности сервера 
	   (если не сработается маршрутизация, то проблемы с сервером);
	    
Разделение по методам запроса происходит из анализа 
$_SERVER['REQUEST_METHOD']

Передача данных:
данные из URL (Query) попадают в массив $_GET независимо
от метода запроса.
Данные из тела попадают в массив $_POST, но, только если 
заголовок Content-Type соответствует html-форму. Иначе,
доступ к телу пакета запроса предоставляет специальный
поток (файл) "php://input".
С ним возможны любые файловые операции чтения, например,
file_get_contents("php://input"), возвращающая весь
контент в виде строки.
Ответные данные передаются выводящими операциями:
echo или print, а также средство разработчика var_dump($_SERVER);
Статус-код ответа устанавливается:
http_response_code(412);
Заголовки:
header("Key: value");

Основные параметры запроса и режимов работы собираются в 
массиве $_SERVER[], его состав зависит от версии Apache, PHP, ОС.
Данные лучше уточнить при конкретной реализации.
